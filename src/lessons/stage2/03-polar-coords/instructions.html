换一种描述位置的方式

我们习惯的直角坐标 (x, y) 说："往右走多少，往上走多少"。但有些图案——比如花瓣、旋涡、靶心——天然是围绕中心展开的。用 (x,y) 描述它们很痛苦。

极坐标 (r, θ) 换了一种说法："离中心多远，朝哪个方向"。

从 (x,y) 转换到 (r,θ)

r = length(uv) — 到中心的距离
θ = atan(uv.y, uv.x) — 方向角（-π ~ π）

这两个值完全等价于 (x,y)，只是描述方式不同。

极坐标的魔力

当你把 sin 的输入从 uv.x（直线方向）换成 θ（角度方向），图案就从"波纹条"变成了"花瓣"：

  • sin(θ * 6.0) → 6 个花瓣（θ 转一圈 2π，sin 做了 6 个完整周期）
  • sin(θ * 3.0) → 3 个花瓣
  • 花瓣的"大小"由 sin 的振幅控制
极坐标能轻松画出什么？
● 花瓣/星形：r = base + amp * sin(θ * N)
● 螺旋：r = a * θ（r 随角度线性增长）
● 齿轮/锯齿：r = base + amp * fract(θ * N / 2π)
● 心形线：r = 1 - sin(θ)（著名的心形曲线！）
💡 中心化很重要！
极坐标的"中心"默认是 (0,0)。我们的 UV 通常是 0~1，所以需要先减去 0.5 把中心移到画面正中间：vec2 uv = gl_FragCoord.xy / u_resolution.xy - 0.5;

🎯 动手试试

1. 改 6.0（花瓣数量）→ 3.0, 5.0, 8.0

2. 让花瓣旋转：把 angle 改成 angle + u_time

3. 改 0.3（振幅）看花瓣大小变化

4. 试心形线：r = 0.5 * (1.0 - sin(angle))
