反三角函数：逆向思维

前面我们学了 cos/sin 把角度变成坐标。但如果反过来——已知坐标 (x, y)，怎么求角度？这就是 atan(y, x) 的工作。

为什么要用 atan(y, x) 而不是 atan(y/x)？

GLSL 提供了两种 atan：

  • atan(y/x)（单参数）→ 只返回 [-π/2, π/2]，无法区分对面象限
  • atan(y, x)（双参数）→ 返回 [-π, π]，覆盖完整 360°
⚠️ 永远用双参数版本！
单参数 atan(y/x) 有两个致命问题：
1. 第一三象限、第二四象限的点会混淆（返回相同角度）
2. 当 x = 0 时会除零崩溃
atan(y, x) 完美解决了这两个问题。

常用套路

  • 获取角度：float angle = atan(uv.y, uv.x); → 范围 [-π, π]
  • 归一化到 [0, 1]：angle / (2.0 * 3.14159) + 0.5
  • 分扇区：floor(angle / (2π) * N) → 得到 0 ~ N-1 的扇区编号
  • 扇区内坐标：fract(angle / (2π) * N)
atan + length = 极坐标变换
float r = length(uv); → 距离
float θ = atan(uv.y, uv.x); → 角度
有了 (r, θ)，你就能画出所有放射状、旋转对称的图案：花瓣、齿轮、星形、万花筒……
💡 atan 的"接缝"问题
atan 在 x 轴负方向（左侧）有一条"接缝"：角度从 +π 突然跳到 -π。
画放射条纹时你可能会在左侧看到一条不自然的线。解决方案：用 fract() 包裹角度值。
