🖼️ 渲染到纹理 — 多 Pass 管线概念

🗂️纹理采样

为什么需要 RTT？

单屏 fragment 课只能看当前像素自己的 UV。真实水面、GPGPU 粒子需要「这一帧写入纹理 → 下一帧读回来」——即 Render to Texture（RTT） 或 FBO。

Pass A：simulate → 写入 texA
Pass B：fragment 里 texture2D(texA, uv) → 更新 → 写入 texB
每帧交换 texA ↔ texB（ping-pong）
本课不做真实 FBO（主课程仍用单屏 ShaderRenderer）。用数学模拟「读上一状态 + 写新状态」：把上一帧结果想成函数 f(uv, t)，本帧在邻域采样它——与下一课「纹理采样」同一思维。
💡 Dracarys 对应
● 水面高度场：每帧更新一张「高度纹理」
● GPGPU 粒子：位置纹 / 速度纹双缓冲
● 后处理：整屏 color buffer → Master pass

🎯 动手试试

1. 用 sin(uv.x*20. + u_time) 模拟「上一帧状态场」

2. 对 uv 加小偏移再采样，可视化「读邻域 = 读纹理」

3. 混合时间：让场随 u_time 扩散（模糊邻域平均）

4. 思考：若真用 FBO，哪一步对应你的 shader？
