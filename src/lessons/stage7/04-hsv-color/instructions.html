🌈 HSV 颜色空间

RGB 的不直观之处

想要"让颜色更鲜艳"或"保持色调不变只改亮度"？在 RGB 中很难做到——三个通道同时影响色调、饱和度和亮度。HSV 把这三个属性分开了。

HSV 三分量

  • H（Hue, 色相）：颜色的"类型"——0°=红, 60°=黄, 120°=绿, 180°=青, 240°=蓝, 300°=紫, 360°=回到红。像色轮。
  • S（Saturation, 饱和度）：颜色的"鲜艳度"。0=灰色, 1=最鲜艳
  • V（Value, 明度）：颜色的"亮度"。0=黑色, 1=最亮

在 Shader 中为什么需要 HSV？

  • 彩虹效果：H 从 0 到 1 线性变化 = 完整彩虹
  • 色温调整：只改 H 偏移一点 = 整体色调偏暖/冷
  • 闪烁效果：只改 V = 亮度脉动，色调不变
  • 去饱和：S 从 1 渐变到 0 = 彩色到黑白的过渡

RGB ↔ HSV 转换

GLSL 没有内置 hsv2rgb！需要自己实现。最简洁的版本：
vec3 hsv2rgb(vec3 c) {
  vec3 p = abs(fract(c.xxx+vec3(0,2./3.,1./3.))*6.-3.);
  return c.z * mix(vec3(1), clamp(p-1.,0.,1.), c.y);
}
只有两行代码，但巧妙地用 fract + abs 构造了 R/G/B 三个三角波相位偏移！
