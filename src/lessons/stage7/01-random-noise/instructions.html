🎲 伪随机与哈希噪声

GPU 上的"随机"

GPU 没有 random() 函数。Shader 在不同帧、不同像素间必须给出相同的结果（确定性）。所以我们使用伪随机——看起来随机但完全可重现的函数。

经典哈希函数

hash(p) = fract(sin(dot(p, vec2(12.9898, 78.233))) × 43758.5453)

这个"魔法数字"组合是 Shader 社区的经典配方：

  • dot(p, vec2(...)) → 把 2D 坐标压缩为 1D 值
  • sin(...) → 制造混乱（sin 的高频部分看起来很"乱"）
  • × 43758.5453 → 放大小数部分的差异
  • fract(...) → 只取小数部分，确保输出在 [0,1)

Value Noise — 最简单的连续噪声

哈希噪声是"每个点独立随机"，看起来像电视雪花。要得到平滑的噪声，需要插值：

  1. 在整数格点上生成随机值
  2. 在格点之间用 smoothstep 做双线性插值
  3. 结果是平滑的、连续的随机起伏——Value Noise
Perlin Noise vs Value Noise：
● Value Noise：在格点上存储随机值，插值。简单但有明显的"方格感"。
● Perlin Noise：在格点上存储随机梯度，用点积插值。更自然，无方格感。
● Simplex Noise：Ken Perlin 的改进版，更快，更少方向偏好。
💡 注意浮点精度！
这个哈希函数在坐标值很大时会失精度（sin 的输入超过几千就不可靠了）。在大世界场景中，先对坐标做 mod 来限制范围。
