📏 abs() 绝对值 与 mod() 取模

📐坐标居中原理 📈smoothstep 原理 🎨mix 混合原理 ❓互动解说本课代码

这两个函数看起来简单，但它们的几何含义才是 Shader 编程的精髓。

abs(x) — "折叠"空间

数学上，abs(x) 就是取绝对值。但在 Shader 的几何世界里，它的作用更直观——沿轴做镜像。

  • abs(-3.0) = 3.0，abs(3.0) = 3.0 — 正负都变正
  • abs(uv.x - 0.5) → 从中心向两侧展开，左右完全对称
  • 直觉：想象把一张纸对折——折线就是 abs 的轴心
abs 在 SDF 中的妙用：
● 画环：abs(distance) - thickness → 取距离的绝对值后减去厚度
● 对称图形：abs(uv.x) → 只需画右半边，左半边自动镜像
● 这个技巧可以节省一半的 SDF 计算！

mod(x, y) — "循环"空间

mod(x, y) 是取模（求余数），但在 Shader 中，它的几何含义是让空间重复循环。

  • mod(5.0, 3.0) = 2.0（5 除以 3，余数 2）
  • 等价于 x - y * floor(x/y)
  • 和 fract 的关系：fract(x) = mod(x, 1.0)（fract 相当于对 1 取模）

棋盘格：mod 的经典应用

怎么判断一个格子是"黑格"还是"白格"？

  • 给每格编号（用 floor 取整），然后看编号的奇偶性
  • mod(cellID.x + cellID.y, 2.0) → 奇数格返回 1，偶数格返回 0
  • 这就是国际象棋棋盘的数学公式！
💡 abs vs mod 的区别
● abs = "折叠"（镜像对称，左右一样）
● mod / fract = "循环"（重复平铺，每段一样）
一个产生蝴蝶翅膀般的对称，一个产生墙纸般的平铺。
⚠️ mod 在负数时的行为：在 GLSL 中，mod(-1.0, 3.0) 返回 2.0（结果总是正数），这和某些编程语言（如 C/C++）不同。如果 UV 坐标经过中心化后可能为负，请注意这个差异。
