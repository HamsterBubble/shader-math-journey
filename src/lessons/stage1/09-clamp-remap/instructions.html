为什么需要"收束"值？

在 Shader 中，数学运算的结果经常会跑出我们期望的范围——颜色值超过 1.0 会过曝，UV 坐标变成负数会产生穿帮。clamp() 就是你的"安全网"。

clamp(x, min, max)

clamp(x, min, max) = min(max(x, min), max)

把 x 限制在 [min, max] 范围内：

  • clamp(1.5, 0.0, 1.0) = 1.0 — 超出上限，截断到上限
  • clamp(-0.3, 0.0, 1.0) = 0.0 — 超出下限，截断到下限
  • clamp(0.7, 0.0, 1.0) = 0.7 — 在范围内，原值不变
💡 saturate(x)
在某些 Shader 语言中（如 HLSL），有个快捷函数 saturate(x) = clamp(x, 0.0, 1.0)。GLSL 中没有这个函数，但你可以自己定义：
#define saturate(x) clamp(x, 0.0, 1.0)

值域重映射 — Shader 的核心技能

更多时候，你需要的不是简单截断，而是将一个范围的数线性映射到另一个范围。比如：把鼠标位置（0~800 像素）映射到颜色（0~1）。

mapped = (x - inMin) / (inMax - inMin) × (outMax - outMin) + outMin

这个公式分三步理解：

  • 第一步：(x - inMin) / (inMax - inMin) → 先归一化到 0~1
  • 第二步：乘以 (outMax - outMin) → 缩放到目标范围宽度
  • 第三步：加上 outMin → 偏移到目标起点
重映射在 Shader 中无处不在：
● sin() 输出 -1~1 → 映射到 0~1 做颜色
● 距离值 0~1 → 映射到温度色阶（蓝→红）
● 不同 SDF 形状的值域 → 统一到相同范围做混合
smoothstep(a, b, x) 本质上就是带平滑曲线的重映射：把 [a, b] 映射到 [0, 1]。
