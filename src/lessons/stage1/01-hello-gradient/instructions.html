🎨 你好，渐变！

Shader 是什么？

想象一下：你站在一面巨大的像素墙前，墙上有几百万个小灯泡。你需要同时告诉每一个灯泡该亮什么颜色。这就是 Shader（着色器）—— 一段运行在 GPU 上的小程序，它为屏幕上的每一个像素并行计算颜色。

关键词是"并行"：CPU 是一个聪明的工人，一次做一件事；GPU 是几千个简单的工人，同时做同一件事。Shader 就是写给这几千个工人的"工作手册"。

Shader 的输入输出非常简单：
输入：当前像素的坐标 gl_FragCoord.xy（单位：像素）
输出：这个像素的颜色 gl_FragColor = vec4(r, g, b, a)
每个分量范围 0.0~1.0，其中 a 是透明度（通常设为 1.0 = 不透明）。

UV 坐标 —— 归一化的像素地址

直接用像素坐标有个问题：在 1920×1080 屏幕上写的代码，换到 800×600 就全变了。

解决方案：归一化。把像素坐标除以屏幕尺寸，映射到 0~1：

vec2 uv = gl_FragCoord.xy / u_resolution.xy;

归一化之后：

  • uv.x = 0.0 → 屏幕最左边
  • uv.x = 1.0 → 屏幕最右边
  • uv.y = 0.0 → 屏幕最下面（注意！和网页相反）
  • uv.y = 1.0 → 屏幕最上面

这样代码就和分辨率无关了——一份代码在任何屏幕上效果一样。

第一个 Shader：坐标变颜色

这节课的骨架代码做了最简单的事：把 uv.x（水平位置）直接当作红色强度。

  • 最左边 uv.x = 0 → vec3(0,0,0) = 纯黑
  • 最右边 uv.x = 1 → vec3(1,0,0) = 纯红
  • 中间 → 从黑到红平滑过渡（渐变）

这就是你的第一个"渐变"，它的本质并不是什么特殊效果——只是坐标的值恰好可以当作颜色用。

💡 核心洞察：Shader 编程的本质就是"用数学函数把坐标映射成颜色"。接下来的所有课程，都是在学不同的映射函数。

🎯 动手试试

1. 把 uv.x 改成 uv.y → 从黑到红变成竖向渐变

2. 第三个分量改成 1.0：vec3(uv.x, 0.0, 1.0) → 加入蓝色

3. 试试 vec3(uv.x, uv.y, 0.5) → 同时用两个坐标控制两个颜色通道

4. 试试 vec3(uv.y, uv.x, 1.0-uv.x) → 更复杂的颜色组合
