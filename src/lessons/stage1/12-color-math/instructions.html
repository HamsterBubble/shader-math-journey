🎨 RGB 颜色的数学本质

颜色就是 vec3

在 Shader 中，颜色没有什么"特殊身份"——它就是一个三分量向量 vec3(r, g, b)，每个分量 0~1。这意味着你对向量做的一切数学运算，都可以用来操作颜色。

加法混色 — 光的叠加

现实中的光是"越加越亮"的（加色混合）：

  • 🔴 + 🟢 = 🟡 黄色
  • 🔴 + 🔵 = 🟣 品红
  • 🟢 + 🔵 = 🔵 青色
  • 🔴 + 🟢 + 🔵 = ⚪ 白色

加法适合表现发光效果：多个灯光叠加、火焰、霓虹灯。

乘法混色 — 滤镜效果

乘法像"透过有色玻璃看世界"——只保留被允许通过的颜色分量：

  • color * vec3(1,0,0) → 红色滤镜，只让红光通过
  • color * 0.5 → 整体变暗（亮度减半）
  • color * lightColor → 彩色光照射物体的效果

乘法适合表现材质着色：光照计算中 光照强度 × 材质颜色 就是乘法。

Cosine 调色板 — 一行代码的无限渐变

最令人惊叹的颜色技巧来自 Shadertoy 大神 Inigo Quilez (IQ)：

color = a + b × cos(2π × (c × t + d))

其中 a, b, c, d 都是 vec3 参数，t 是 0~1 的输入值。

  • a — 基础色（颜色中点）
  • b — 振幅（颜色变化范围）
  • c — 频率（颜色变化次数）
  • d — 相位（每个通道的偏移，决定色相变化）
IQ 调色板的威力：仅仅调整 4 个 vec3 参数，就可以生成从火焰到海洋、从日落到极光的任意渐变色。几乎每个 Shadertoy 作品都在用这个技巧。

推荐网站：iquilezles.org/articles/palettes 有详细解释和交互式工具。
💡 为什么用 cos 而不是任意函数？
因为 cos 的输出永远在 -1~1 之间，确保颜色不会溢出。加上 a 和 b 的控制后，可以精确地把颜色限制在合理范围内。
