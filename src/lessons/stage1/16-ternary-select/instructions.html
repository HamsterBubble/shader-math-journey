🔀 条件选择：三元替代

GPU 不喜欢 if-else

在 CPU 上我们习惯用 if (x > 0) { A } else { B }。但在 GPU 上，分支（branching）会严重影响性能——因为同一 Warp/Quad 的线程必须走相同的路径。

无分支选择三件套

  • step(edge, x)：x ≥ edge 返回 1.0，否则 0.0 → 作为 "mask"
  • mix(a, b, t)：用 t 在 a 和 b 之间混合
  • 组合：mix(A, B, step(threshold, value)) = 无分支三元运算
// 有分支（避免）
if (x > 0.5) color = red; else color = blue;

// 无分支（推荐）
color = mix(blue, red, step(0.5, x));

多条件叠加

需要多个区域不同效果？用多个 mask 叠加：

  • 每个 step 生成一个 0/1 的 mask
  • 用 mix 或乘法叠加：color = mix(color, newColor, mask)
  • 后面的 mix 覆盖前面的——像 Photoshop 的图层
⚠️ 何时可以用 if
如果条件对整个画面是统一的（如 if (u_time > 5.0)），分支没问题——所有像素走同一路。
只有当条件因像素而异（如 if (uv.x > 0.5)）时，才需要替换为无分支写法。
