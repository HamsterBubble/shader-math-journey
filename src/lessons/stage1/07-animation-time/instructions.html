⏱️ u_time — 让一切动起来

时间：Shader 的第四维

到目前为止，我们的画面都是静态的。但 Shader 有一个秘密武器：u_time——一个从程序启动开始不断递增的浮点数（单位：秒），它是一切动画的源泉。

sin(u_time)：永不停歇的摆钟

sin() 函数天生就是做动画的——它的输出在 -1 和 1 之间永远来回振荡，永远不会停。配合 u_time：

  • sin(u_time) → 输出在 -1 ~ 1 之间来回
  • sin(u_time) * 0.5 + 0.5 → 映射到 0 ~ 1（最常用！）
  • sin(u_time * 2.0) → 2 倍速振荡
  • sin(u_time * 0.5) → 慢动作振荡
💡 万能动画公式
sin(u_time) * 0.5 + 0.5 是 Shader 中出现频率最高的代码片段之一。它把 sin 的 -1~1 映射成更实用的 0~1，可以用于：控制颜色、控制大小、控制位置……几乎任何需要"来回变化"的场景。

让空间随时间变化

动画的本质是：把 u_time 混入坐标运算。几种常用手法：

  • 平移：uv.x + u_time * 0.1 → 画面缓缓滚动
  • 脉动：sin(length(uv)*10.0 - u_time*3.0) → 波纹从中心扩散
  • 扭曲：uv.x + sin(uv.y*10.0 + u_time)*0.05 → 画面像水面一样摆动
关键理解：Shader 没有"帧"的概念。每帧执行一次 main()，u_time 的值稍微增加一点。由于 sin 是连续函数，这些微小的变化产生了平滑的动画效果。GPU 不是在"播放动画"，而是在每帧重新计算整幅画面。
⚠️ 注意精度：u_time 会无限增长。当它变得很大（如超过 10000）时，sin(u_time * 100.0) 中的大数乘法可能产生浮点精度问题。实际项目中常用 mod(u_time, 6.28) 限制范围。

1. 改 sin 里的速度乘数（0.5 = 慢，5.0 = 快）

2. 方向改为 uv.x + uv.y → 斜向波纹

3. 扭曲效果: uv.x + sin(uv.y*10.0+u_time)*0.05

4. 试试 abs(sin(u_time)) → 只取正值，产生"弹跳"效果
