🔲 step() — 阶跃函数

数字世界的"开关"

生活中有很多是/否判断：灯是开的还是关的？温度超过 100°C 水就沸腾。这种"到了某个边界就突然切换"的逻辑，在数学上叫阶跃函数。

step() 的行为

step(edge, x) = x < edge ? 0.0 : 1.0
  • 当 x < edge 时 → 返回 0.0（"关"）
  • 当 x >= edge 时 → 返回 1.0（"开"）

就像一个跷跷板：超过中心点就咔嗒一声翻过去，没有中间状态。

为什么 Shader 用 step() 而不用 if-else？

在普通编程中，我们写 if (x > 0.5) { ... }。但 GPU 有个特性：

GPU 的并行本质：GPU 上几千个核心必须走完全相同的代码路径。如果遇到 if-else 分支，两种情况的代码都会执行，然后丢弃不要的那个——浪费了一半算力！

step() 用纯数学实现了同样的效果：乘以 0 就等于"不要"，乘以 1 就等于"保留"，但没有分支。这在 GPU 上效率远高于 if-else。

实际用途

  • 分割画面：step(0.5, uv.x) → 屏幕左黑、右白
  • 制作遮罩：圆内 = 1、圆外 = 0，用 step 实现
  • 组合形状：多个 step 相乘 = AND（交集），相加 = OR（并集）
⚠️ 注意参数顺序！step(edge, x) 不是 step(x, edge)。第一个参数是阈值，第二个是要判断的值。搞反了不会报错，但效果完全反转。

🎯 动手试试

1. 修改 0.5 阈值 → 看分界线移动

2. 改成 step(0.5, uv.y) → 上下分割

3. 两个 step 相乘: step(0.3, uv.x) * step(0.3, uv.y) → 只有右上角亮

4. 用 1.0 - step(0.7, uv.x) → 反转效果（左亮右暗）
