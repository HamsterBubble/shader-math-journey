🏔️ 从高度图推算法线

高度图 → 3D 表面

一张灰度图可以当作"高度图"——亮的地方高、暗的地方低。虽然它只是 2D，但通过计算导数，我们可以推算出每个点的3D 法线方向。

核心公式

dx = height(x+ε, y) - height(x-ε, y)
dy = height(x, y+ε) - height(x, y-ε)
normal = normalize(vec3(-dx, -dy, ε*2.0))

原理：在 x 方向采样两个相邻点的高度差作为 x 方向的斜率（导数），y 方向同理。然后用叉积构造法线。

为什么是 "中心差分"？

我们用 (x+ε) - (x-ε) 两侧的差，而不是 (x+ε) - x 单侧的差。前者叫中心差分，精度是后者的两倍，且结果更对称。

这就是法线贴图的原理！
游戏中的法线贴图 (Normal Map) 就是预先计算好的法线存储为 RGB 图像：
● R = 法线的 x 分量
● G = 法线的 y 分量
● B = 法线的 z 分量
蓝紫色的法线贴图（z 分量最大 = 朝上）是游戏开发者最熟悉的颜色。
💡 ε 的选择
ε 太大 → 法线过于平滑，丢失细节
ε 太小 → 受像素精度限制，产生噪声
通常 1~3 像素宽度是好的选择：ε = 1.0/u_resolution.x

🎯 动手试试

1. 用 sin 波生成高度图，从中推算法线

2. 把法线值映射到颜色（看到蓝紫色法线图）

3. 用推算出的法线做 Lambert 光照

4. 改 ε 值看法线的平滑度变化
