✨ 抗锯齿 (Anti-Aliasing)

为什么有锯齿？

像素是方形的，但边缘可能是斜的或弯的。当一条斜线穿过像素网格时，每个像素只能选择"画"或"不画"——这种硬切割产生了阶梯状的锯齿（aliasing）。

Shader 抗锯齿的核心思想

别做硬决定（非黑即白），而是了解"这条边在这个像素中覆盖了多大面积"，按比例混合颜色。

fwidth 自适应抗锯齿

这是 Shader 中最实用的抗锯齿技巧：

float fw = fwidth(d);
float edge = smoothstep(-fw, fw, d);

fwidth(d) 告诉我们 d 在一个像素中变化了多少。把这个值作为 smoothstep 的过渡宽度——恰好模糊一个像素的范围。

为什么"恰好一个像素"是正确的？

  • 模糊太多 → 边缘变软，丢失锐利感
  • 模糊太少 → 仍有锯齿
  • fwidth 自动适应缩放：放大时模糊范围变小（像素覆盖的空间更小），缩小时变大
SDF + fwidth = 完美组合
SDF 天生就有精确的"到边界的距离"信息。配合 fwidth，一行代码就能得到像素级完美的抗锯齿边缘——这是 SDF 渲染文字的优势所在（距离场字体）。
💡 对比测试
最好的验证方式：半屏用 step(0.0, d)（有锯齿），半屏用 smoothstep(-fw, fw, d)（抗锯齿），放大对比效果。

🎯 动手试试

1. 对比 step 和 smoothstep+fwidth 的边缘质量

2. 把 fw 乘以 2.0 或 0.5 看"过度模糊"和"不够模糊"

3. 在高频图案（密集条纹）上测试抗锯齿效果

4. 画一条倾斜的线，对比有/无抗锯齿的差异
