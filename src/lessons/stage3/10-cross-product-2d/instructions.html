叉积不该是 3D 的吗？

3D 叉积返回一个垂直向量。2D 没有"垂直于屏幕"的空间，所以 2D 叉积返回一个标量——但这个标量信息量极大。

cross2d(a, b) = a.x × b.y - a.y × b.x

2D 叉积的三大用途

  • 方向判定：结果 > 0 → b 在 a 的左侧；< 0 → 右侧；= 0 → 共线
  • 有向面积：|cross(a,b)| = 两向量张成的平行四边形面积
  • 到直线距离：点 p 到向量 v 所在直线的有符号距离 = cross(normalize(v), p)

在 Shader 中的重要场景

判断"在三角形内部"
对三角形 ABC 的三条边分别算叉积 cross(AB, AP), cross(BC, BP), cross(CA, CP)。
如果三个结果同号→ 点在三角形内。这就是 GPU 光栅化的核心算法！
💡 GLSL 没有 2D cross
GLSL 的 cross() 只接受 vec3。2D 叉积需要自己写：
float cross2d(vec2 a, vec2 b) { return a.x*b.y - a.y*b.x; }
