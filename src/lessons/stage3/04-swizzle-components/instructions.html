🔀 Swizzle — 分量魔法

GLSL 的独门绝技

在大多数编程语言中，访问数组元素用下标：arr[0]。GLSL 发明了一种更优雅的方式——Swizzle（分量重排）：用点号加字母直接提取、重组向量的分量。

四组等价的访问名称

  • .xyzw — 坐标语义（位置、方向）
  • .rgba — 颜色语义（红绿蓝透明）
  • .stpq — 纹理坐标语义（较少用）

三组字母完全等价：v.x == v.r == v.s。选哪种只是为了代码可读性。

Swizzle 的威力

  • 提取：v.xy → 从 vec4 取前两个分量得 vec2
  • 交换：v.yx → 交换 x 和 y（镜像翻转！）
  • 复制：v.xxx → 三个分量都用 x → 灰度化
  • 重排颜色：color.bgr → 蓝红互换（"胶片负片"感）
  • 升维：vec3(v.xy, 0.0) → 2D 向量扩展为 3D
实际应用场景：
● 灰度化：float gray = dot(color.rgb, vec3(0.299, 0.587, 0.114)); 然后 vec3(gray)
● 通道分离：color.r 单独控制某个效果的强度
● 色温调整：color *= vec3(1.2, 1.0, 0.8) → 暖色调
● 坐标翻转：uv.yx → 90 度旋转
💡 Swizzle 是零成本的！
GPU 硬件层面原生支持 Swizzle——它不是"拷贝"操作，而是"重新路由"。所以 v.yzx 和 v.xyz 性能完全一样。

🎯 动手试试

1. 把 .rgb 改成 .brg 或 .gbr → 色彩变换

2. 试 color.rrr → 只取红色通道（灰度）

3. 加一行 color *= vec3(1.2, 0.9, 0.8); → 暖色温调整

4. 用 uv.yx 替代 uv → 整个画面旋转 90°
