不可交换——矩阵乘法的本质

在普通数学中，3 × 5 = 5 × 3。但矩阵乘法不可交换：A × B ≠ B × A。这意味着变换的顺序决定最终结果。

直觉理解

想象你要先转身 90°，再向前走 3 步。和先向前走 3 步、再转身 90°，最终位置完全不同！

GLSL 中的变换顺序

矩阵乘法从右到左应用：

result = M2 × M1 × v → 先应用 M1，再应用 M2
  • scaleMat * rotateMat * uv → 先旋转，后缩放
  • rotateMat * scaleMat * uv → 先缩放，后旋转

SRT 标准顺序

游戏引擎几乎都用这个默认顺序：

  1. Scale（缩放）— 先调整大小
  2. Rotate（旋转）— 再转动方向
  3. Translate（平移）— 最后移到位置

在代码中：TranslateMat * RotateMat * ScaleMat * point（从右往左读 = SRT）

绕任意点旋转的三步法：
想绕点 P 旋转，而非绕原点？
1. 先平移 -P（把 P 移到原点）
2. 旋转
3. 平移 +P（移回去）
代码：uv -= pivot; uv = rotate(uv); uv += pivot;
⚠️ Shader 中的"反向"问题
在 Shader 中，我们变换的是 UV 坐标，不是物体。所以：
● 放大 UV = 图案缩小
● 旋转 UV 顺时针 = 图案逆时针旋转
如果想直觉地操作，可以对矩阵求逆，或直接用负角度/倒数缩放。
