✨ GPGPU 粒子 — 用纹理当粒子数据库

CPU 循环更新一万颗粒子？太慢。GPGPU 思路：把每颗粒子的状态存进纹理的一个 texel，用全屏 Quad + Fragment Shader并行更新——GPU 天生擅长这个。

一、纹理 = 粒子数组

一张 256×256 的纹理可以存 65536 颗粒子。每个 texel 的 RGBA 通道分工明确：

通道Pass 1 速度Pass 2 位置
Rvel.xpos.x
Gvel.ypos.y
Bvel.zpos.z
A—life

二、双 Pass 更新循环

Pass 1 — 速度（受力积分）

vec3 spring = (anchor - pos) * uSpringK;   // 弹簧：拉回模型表面
vec3 damp   = -vel * uDamping;             // 阻尼：能量衰减
vec3 mouse  = normalize(pos - mousePos)
            * uRepel / dist(pos, mousePos); // 鼠标推开
vel += (spring + damp + mouse) * dt;

Pass 2 — 位置（欧拉积分）

pos += vel * dt;
gl_FragColor = vec4(pos, life);
Ping-Pong 双缓冲：Pass 1 读 texA 写 texB，Pass 2 读 texB 写 texA，每帧交换。

四、Dracarys 龙身粒子（dracarys-05-gpgpu-particles）

案例在龙模型表面撒数千锚点，粒子被弹簧约束在鳞片上，鼠标 NDC 坐标产生径向斥力。视觉上龙身像有生命般起伏——这是 GPGPU 弹簧-阻尼系统的典型应用。
