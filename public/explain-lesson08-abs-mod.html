❓ 代码彻底拆解：abs() 与 mod()

这节课引入了两个看似简单的数学函数——abs() 和 mod()——但它们在 Shader 的空间操作中有着令人震撼的几何意义：一个是折叠纸张，一个是无限复印。

第一步：坐标中心化

uv = uv * 2.0 - 1.0;
uv.x *= u_resolution.x / u_resolution.y;

把 uv 从 [0,1] 变成 [-1,1]，让屏幕中心变成 (0,0)。第二行修正宽高比，防止画面被挤压。

这是 SDF 和几何操作的标准起手式，我们在第五课（画圆）已经学过。

第二步：abs() — 折纸术

vec2 mirror = abs(uv);

abs() 做了什么？它把所有负数变成正数。在二维坐标系中，这意味着：

  • 左半边（uv.x < 0）翻折到右半边
  • 下半边（uv.y < 0）翻折到上半边
  • 四个象限的坐标全部挤进了第一象限（右上角）

第三步：用 mirror 坐标画 V 形线

float v_shape = smoothstep(0.02, 0.0, abs(mirror.y - mirror.x * 0.8) - 0.01);

这行代码看起来很复杂，我们把它拆成 5 个子步骤，每步配一张图。鼠标悬停可以检查每个像素在这一步的值。

3.1 — mirror.x * 0.8 → 一条直线的方程

在数学中，y = 0.8x 定义了一条从原点出发、斜率为 0.8 的直线。
因为我们用的是 mirror（经过 abs 折叠的坐标，x 和 y 都在 [0,1] 范围内），所以这条线只存在于右上角。

绿线 = y=0.8x（在 mirror 空间里）

为什么选 0.8？

  • 斜率 0.8 表示"x 走 1 单位，y 走 0.8 单位"
  • 斜率 1.0 → 45° 对角线
  • 斜率 0.8 → 比 45° 稍平一些，视觉上更像一个"V"而不是"X"

3.2 — mirror.y - mirror.x * 0.8 → 带符号的"距离"

对每个像素，用它的 y 坐标减去"直线上同 x 位置的 y 值"：

  • 结果 > 0 → 这个点在直线上方（y 比直线的 y 大）
  • 结果 < 0 → 这个点在直线下方
  • 结果 = 0 → 恰好在直线上

绿色=正值（线上方），红色=负值（线下方），黑色分界线=直线本身

3.3 — abs(mirror.y - mirror.x * 0.8) → 无符号距离

abs() 把负值翻正，现在上方和下方统一变成"到直线的距离"：

  • 直线上 → 距离 = 0（最亮/最近）
  • 离直线越远 → 距离越大（越暗/越远）
  • 不再区分上方还是下方 → 直线两侧对称

亮度 = 距离。越靠近直线越亮，两侧完全对称

3.4 — abs(...) - 0.01 → 挖出一条"沟"

减去 0.01 意味着：

  • 距离 < 0.01 的像素 → 减后变负数（在线条内部，沟底）
  • 距离 = 0.01 的像素 → 减后刚好 = 0（线条边缘）
  • 距离 > 0.01 的像素 → 减后仍为正（在线条外部）

0.01 就是这条线的半宽。全宽 = 0.02 个坐标单位。

青色区域 = 值 ≤ 0（线条内部及附近），暗色 = 值 > 0（线条外部）

3.5 — smoothstep(0.02, 0.0, ...) → 柔化边缘出最终线条

smoothstep(0.02, 0.0, x) 的含义是：

  • 当 x ≤ 0.0 → 输出 1.0（全亮，在线条核心）
  • 当 x ≥ 0.02 → 输出 0.0（全暗，远离线条）
  • 当 0.0 < x < 0.02 → 平滑过渡（柔和边缘，anti-aliasing）

注意参数是反向的（0.02 → 0.0），所以是"越小越亮"。这正好利用了 step 3.4 中"负值/小值=在线条内"的特征。

最后，因为生活在 mirror 空间中，右上角的一条线通过 abs 镜像成了四条线，组成 X / V 形。

最终 v_shape 值：粉色亮线 = 值接近 1，黑色 = 值为 0

💡 整条流水线总结
mirror.x * 0.8 → 定义一条斜线
mirror.y - ... → 计算到斜线的带符号距离
abs(...) → 去掉符号，两侧对称
... - 0.01 → 给线条设置宽度（0.01 = 半宽）
smoothstep(0.02, 0.0, ...) → 把距离转化为亮度（近=亮；远=暗）
abs(uv)（第二步的 mirror） → 一条线自动变成四条线！

第四步：mod() — 复印机

float stripes = mod(uv.x * 8.0 + u_time, 2.0);
float stripe_mask = step(1.0, stripes);

mod(x, 2.0) 让 x 每走过 2 就归零重来。搭配 step(1.0, ...) 变成 0/1 交替的方波：

🔍 mod + step = 奇偶条纹

8
2.0

背后原理：uv.x * 8 把坐标从 0~1 放大到 0~8。mod(..., 2.0) 让它在 0~2 之间循环 4 次。step(1.0, ...) 在值 ≥1 时输出 1，<1 时输出 0 → 黑白交替条纹！

第五步：棋盘格 — mod 的二维经典

vec2 grid = floor((uv + 1.0) * 4.0);
float checker = mod(grid.x + grid.y, 2.0);

棋盘格原理拆解

  • ① (uv + 1.0) — uv 是 [-1,1]，加 1 变成 [0,2]（全正数）
  • ② * 4.0 — 放大到 [0,8]，屏幕被分成 8×8 = 64 格
  • ③ floor(...) — 取整，给每格一个整数编号 (col, row)
  • ④ mod(col + row, 2.0) — 列号+行号的奇偶性：奇数格=1，偶数格=0

就像国际象棋：行列编号之和为奇数就是黑格，偶数就是白格。

第六步：三层叠加合成

color = mix(color, vec3(0.15, 0.05, 0.2) + vec3(0.2) * checker, 0.5);
color = mix(color, vec3(0.3, 0.6, 1.0) * stripe_mask, 0.3);
color += vec3(0.9, 0.3, 0.5) * v_shape;

三层从底到上：

层级内容混合方式比例
① 底层棋盘格（暗紫+亮紫交替）mix(..., 0.5)50%
② 中层蓝色条纹（随时间滚动）mix(..., 0.3)30%
③ 顶层粉色 V 形线（abs 镜像）color +=叠加发光

注意第三层用的是 +=（加法叠加）而不是 mix → 这让 V 形线像发光体一样明亮地叠在画面上，而不是温和地"混入"。
