📏 为什么 uv.x *= u_resolution.x / u_resolution.y？

归一化并居中之后，坐标范围是 [-1, 1] × [-1, 1]——看起来是正方形，但屏幕本身往往是长方形。不修正的话，用 length(uv) 画出来的「圆」会被拉成椭圆。

一、问题从哪来？

假设画布是 800 × 400（宽是高的 2 倍）：

  • 归一化后，横向走满屏幕 uv.x 从 -1 到 1，对应 800 像素
  • 纵向走满屏幕 uv.y 从 -1 到 1，对应 400 像素

也就是说：在坐标里横向走 1 单位，实际跨越的像素是纵向的 2 倍。length(uv) < 0.4 在数学上是圆，在屏幕上横向被拉宽，变成扁椭圆。

宽高比 aspect = u_resolution.x / u_resolution.y
16:9 屏幕 → aspect ≈ 1.78（偏宽）
手机竖屏 9:16 → aspect ≈ 0.56（偏高）

二、修正原理：只拉 X 轴

uv = uv * 2.0 - 1.0;                      // 居中到 [-1,1]
uv.x *= u_resolution.x / u_resolution.y;  // 把 X 轴按宽高比拉伸

乘以 aspect 后，横向 1 个坐标单位对应的物理距离，和纵向对齐了。此时 length(uv) 衡量的是真实的等比例距离，圆就是正圆。

屏幕aspectuv.x 乘以 aspect 后效果
800×4002.0横向范围变成 [-2, 2]圆恢复正圆
1920×1080≈1.78横向略拉宽圆恢复正圆
600×6001.0乘以 1，不变正方形屏无需修正

为什么只改 x 不改 y？两种写法等价，业界习惯拉伸较短的那根轴到与较长轴对齐。宽屏就拉 X，竖屏就… 仍拉 X（此时 aspect < 1，相当于压缩 X）。统一写 uv.x *= aspect 即可适配所有比例。

三、完整起手式（画圆 / SDF 必备）

vec2 uv = gl_FragCoord.xy / u_resolution.xy;  // [0,1]
uv = uv * 2.0 - 1.0;                          // 中心变原点
uv.x *= u_resolution.x / u_resolution.y;      // 等比例空间
float d = length(uv) - 0.4;                  // 正圆 SDF

前三行合称 「NDC + 宽高比校正」，从第 5 课画圆开始会反复出现。详见 坐标居中原理。
