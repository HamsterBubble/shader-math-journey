为什么 uv × 2 − 1 能把原点居中？

一步步可视化 UV 坐标变换的数学原理

一、先看结果：变换前 vs 变换后

下面两个画面展示了同一个像素网格，左边是原始坐标，右边是变换后的坐标：

变换前: uv = fragCoord / resolution
原点在左下角，范围 [0, 1]
→
变换后: uv = uv × 2 − 1
原点在正中央，范围 [-1, 1]

二、数轴上的映射（一维理解）

先别想二维，只看 一条数轴。问题简化为：

怎么把 [0, 1] 变成 [-1, 1]？

[0, 1] × 2 → [0, 2]

乘以 2 是一个 拉伸 操作。想象一根 1 米长的橡皮筋，拉到 2 米长：

原始值× 2 后变化
00没动（左端固定）
0.250.5被拉远了
0.51.0原来的中点变成了 1
0.751.5被拉远了
1.02.0右端从 1 拉到 2

范围变了，但 左端还黏在 0 上，原点还没居中。

[0, 2] − 1 → [-1, 1]

减 1 是一个 平移 操作。整根数轴往左拖 1 个单位：

× 2 后− 1 后含义
0-1左端变成 -1
0.5-0.5
1.00 ← 🎯原来的中点变成了 新原点！
1.50.5
2.01右端变成 +1

关键发现：原来 uv = 0.5 的地方（画面正中间），现在变成了 0！

三、为什么原点在中间？数学证明

问题：画面正中间的像素，原始 UV 值是多少？

在 800×600 的画布上，中间像素坐标是 (400, 300)：

uv = (400, 300) / (800, 600) = (0.5, 0.5)

代入变换公式：

uv = 0.5 × 2.0 − 1.0 = 1.0 − 1.0 = 0.0 ← 变成了 0！原点！

所以 × 2 − 1 的本质是：让原来的 0.5（中点）变成新的 0（原点）。

💡 更通用的理解：线性映射公式

y = x × 2 − 1 其实是一条 直线方程（一次函数）：

y = 2x − 1

这是初中数学：斜率 = 2，截距 = -1。它完成的映射是：

输入 x ∈ [0, 1] → 输出 y ∈ [-1, 1]
x=0.5 时 y=0（原点！）
通用公式:
y = (x−a) / (b−a) × (D−C) + C
把 [a,b] 映射到 [C,D]
把 [0,1] → [-1,1] 代入通用公式：
y = (x−0)/(1−0) × (1−(−1)) + (−1)
= x × 2 − 1 ✅

四、从一维到二维

二维其实就是 x 和 y 各做一次同样的变换，互不影响：

// 原始： uv = gl_FragCoord.xy / u_resolution.xy; // x ∈ [0,1], y ∈ [0,1] // x 和 y 分别做 ×2−1： uv = uv * 2.0 - 1.0; // 等价于： // uv.x = uv.x * 2.0 - 1.0; x: [0,1] → [-1,1] // uv.y = uv.y * 2.0 - 1.0; y: [0,1] → [-1,1]

GLSL 中 vec2 * 2.0 会自动对 x 和 y 分别乘 2，- 1.0 也分别减 1。这就是向量运算的便利。

六、为什么画圆需要原点居中？

原点在左下角
length(uv) - 0.4
圆心跑到角落了 😅
→
原点在中央
length(uv) - 0.4
圆心正好在中间 ✅

💡 直觉理解

length(uv) 计算的是 到原点 (0,0) 的距离。

所以原点在哪，圆心就在哪。× 2 − 1 把原点移到画面中央，圆自然画在中央。

如果你想把圆画到别处，比如右边：length(uv - vec2(0.3, 0.0)) — 本质是修改"距离的参考点"。

七、记忆口诀 🧠

×2 拉伸，−1 平移

[0, 1] ×2 → [0, 2] −1 → [-1, 1]

拉伸让范围够宽 (宽度从 1 变为 2)
平移让中点对齐零 (0.5 → 1.0 → 0.0)
