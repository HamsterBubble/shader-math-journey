⭕ SDF 有符号距离函数：Shader 画图形的灵魂

在 Shader 中，我们不像 Canvas 那样调用 drawCircle()。我们是给屏幕上的每一个像素打上一张距离考卷，然后根据分数决定哪些像素属于图形内部、哪些是外部。这就是 SDF。

一、核心概念：像素到边界还有多远？

SDF = Signed Distance Function (有符号距离函数)。对屏幕上的每个像素，SDF 会计算出一个数字：它离图形边界有多远。

"有符号"这三个字至关重要，因为这个距离有正负方向：

d < 0

像素在内部
（负数 = 被图形包裹）

d = 0

正好踩在边界上
（零 = 擦线分割点）

d > 0

像素在外部
（正数 = 图形之外）

一个数字同时告诉你两件事：在哪一边（符号）和 离边线有多远（绝对值）。这就是 SDF 的全部秘密。

二、最简单的例子：圆的 SDF

float d = length(uv) - radius;

length(uv) 是这个像素到坐标原点(0,0)的物理距离（勾股定理）。减去圆的半径后：

像素在哪length(uv)- radius(0.4)d 值含义
圆心正中0.0- 0.4-0.4深藏在圆的核心
靠近内边缘0.38- 0.4-0.02刚入门，马上到边界了
边界上0.4- 0.40.0正好踩线！
刚出边界0.42- 0.4+0.02刚出门一步
远离圆0.8- 0.4+0.4已经跑到很远了

四、从距离到颜色（SDF 怎么变成画面）

有了 SDF 值 d，你需要把"正负数值"翻译成"屏幕颜色"。以下是四种翻译策略：

策略 ① 硬边裁切 — step(0.0, -d)

d<0 就是 1(白)，d>0 就是 0(黑)。非黑即白，像一个像素级的饼干模具。有明显的锯齿！

策略 ② 柔边抗锯齿 — smoothstep(0.01, -0.01, d)

在 d=0 附近给一点点模糊区间，边缘像素就不再是 0 或 1 的二元对立，而是某个 0.x 的中间值。这就实现了内建抗锯齿！这是最常用的。

策略 ③ 外发光 — exp(-abs(d) * k)

利用指数衰减函数，让边界最亮、向两边极速消散。做出类似灯管或霓虹灯的光晕效果。

策略 ④ 可视化距离场 — 直接用 d 映射颜色

不做任何裁切，直接把 d 的数值当做颜色来渲染，就能看到当前 SDF 的"地形图"——一圈圈的等高线。这在调试复杂图案时非常有用！

五、SDF 的超能力：布尔运算

SDF 最疯狂的好处在于：你可以像搭积木一样组合图形，并且只需要 min 和 max！

💡 图形组合公式卡

  • 并集（合并多个图形）：d = min(sdf1, sdf2) → 哪个更近就选哪个
  • 交集（只保留重叠区）：d = max(sdf1, sdf2) → 必须两个都在内部
  • 差集（挖洞镂空）：d = max(sdf1, -sdf2) → 用 sdf2 把 sdf1 掏空
  • 描边（只画轮廓）：d = abs(sdf) - thickness
  • 圆角（给方形磨角）：d = sdf - roundness
