📐 矩阵变换 — Shader 里"扭曲空间"的钥匙

在 Shader 中，我们不是移动图形本身，而是移动整个坐标空间。矩阵就是执行这种空间扭曲的数学工具。

一、为什么要用矩阵？

如果你想在 Shader 中旋转一个图案，最直觉的做法可能是去改每个像素的颜色。但更高效的方法是：在计算颜色之前，先把 UV 坐标旋转一下！图案自然就跟着转了。

矩阵让你可以把旋转、缩放、错切等多种变换打包成一次乘法：uv = mat2 * uv; 一行搞定！

二、最常用的三种变换

🔄 旋转 (Rotation)

围绕坐标原点旋转角度 θ：

mat2 rot = mat2(
    cos(θ), -sin(θ),
    sin(θ),  cos(θ)
);
uv = rot * uv;

这个 2×2 矩阵把 (x,y) 按照圆周公式重新分配。cos 控制"保留原方向的分量"，sin 控制"注入垂直方向的分量"。两者配合就产生了旋转。

📏 缩放 (Scale)

mat2 scl = mat2(
    sx,  0.0,
    0.0, sy
);
uv = scl * uv;

对角线上的 sx 和 sy 分别拉伸 X 和 Y 轴。sx=2 相当于 X 方向拉伸两倍（图案变扁窄），sy=0.5 把 Y 压缩一半。

⚠️ 注意：这是变换坐标空间！所以 sx=2 会让图案看起来缩小，因为 UV 空间变大了。

↕️ 平移 (Translation)

平移最简单，不需要矩阵，直接加减就行：

uv = uv - vec2(offsetX, offsetY);

减去偏移量 = 图案向那个方向移动。注意是减而不是加！因为我们移的是坐标空间，方向是反的。

四、黄金法则：变换的顺序很重要！

⚠️ 先平移再旋转 ≠ 先旋转再平移

矩阵乘法不可交换！就像现实中：先往前走10步再右转90° 和 先右转90°再往前走10步，你最终站的位置完全不同！

在 Shader 中最常见的顺序是：

  1. 居中：uv = uv * 2.0 - 1.0; 让原点在中间
  2. 旋转：uv = rot * uv; 围绕中心旋转
  3. 缩放：uv *= scale;
  4. 平移：uv -= offset;
