🗂️ texture2D — 用纹理当「GPU 内存条」

单屏 Fragment Shader 只能看当前像素。但把上一帧结果写入纹理，下一 pass 用 texture2D(sampler, uv) 读取——就能实现跨像素、跨帧的状态传递。

一、采样在做什么？

给定 UV 坐标 (0~1)，GPU 在纹理上双线性插值取出颜色。每个 texel 可以存任意数据：RGB 颜色、高度、速度、粒子位置……

vec4 texel = texture2D(uTexture, uv);
UV 超出 [0,1] 时由 wrapS/wrapT 决定：Repeat / Clamp / Mirror

二、邻域采样：水面与 GPGPU 的核心

波动方程、粒子模拟都需要读邻居像素的值。偏移 UV 一个 texel 即可：

vec2 cell = 1.0 / uResolution;   // 一个像素对应的 UV 步长

vec4 center = texture2D(uTex, uv);
vec4 north  = texture2D(uTex, uv + vec2(0.0,  cell.y));
vec4 south  = texture2D(uTex, uv + vec2(0.0, -cell.y));
vec4 east   = texture2D(uTex, uv + vec2( cell.x, 0.0));
vec4 west   = texture2D(uTex, uv + vec2(-cell.x, 0.0));

纹理通道分工（Dracarys 水面）

  • .r — 当前高度 h
  • .g — 上一帧速度 vel
  • .b — 法线 X 分量
  • .a — 法线 Y 分量

四、Dracarys 多 Pass 管线

Ping-Pong 双缓冲

案例 dracarys-03-water 每帧：

  1. 读 texA 的四邻域 → 算新高度 → 写入 texB
  2. 交换 A/B，下一帧反过来
  3. 渲染 pass 采样最终纹理做折射/反射
// 波动更新（简化）
float avg = (north.r + south.r + east.r + west.r) * 0.5;
float newH = avg - center.g;          // 四邻平均 − 旧速度
float newVel = newH - center.r;       // 新速度 = 高度差
gl_FragColor = vec4(newH, newVel, normal.xy);
