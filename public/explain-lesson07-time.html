❓ 代码彻底拆解：u_time 波浪动画

这节课终于让画面动起来了！核心武器就是 u_time——一个从 0 开始不停地增长的数字（单位：秒）。配合 sin() 的永不停歇的震荡，我们能让颜色像海浪一样来回翻滚。

第一步：时间是什么？

uniform float u_time;

这行声明了一个叫 u_time 的变量。它不是我们自己写死的数字，而是外部程序每帧自动更新传进来的"秒表读数"。

程序运行时间u_time 值大白话
刚启动0.0秒表归零
1 秒后1.0指针走了一格
3.5 秒后3.5一直在涨…
100 秒后100.0永远不停！

关键理解：GPU 不是在"播放动画"。它每一帧都完全从零开始重新计算整幅画面，只不过 u_time 的值比上一帧大了那么一丁点（比如 0.016 秒），所以画面看起来就在"动"。

第二步：sin 把永恒增长变成永恒震荡

float wave = sin(uv.x * 10.0 + u_time * 2.0) * 0.5 + 0.5;

这行是整节课的灵魂！让我们拆成四个齿轮：

齿轮 ① uv.x * 10.0 — 空间频率

uv.x 从屏幕左边的 0 到右边的 1。乘以一个数字（频率）后，决定了屏幕上能装下几个正弦波。

10.0

背后原理：乘的数字越大，相当于把空间坐标压缩得越厉害，所以屏幕里能塞进更多个周期。波纹变得更密集。

齿轮 ② + u_time * 2.0 — 时间驱动滚动

这是让波浪"动起来"的关键！sin 函数里加上一个不断变大的数字，就等于把整个波形不断往左推。

2.0

背后原理：u_time 本质就像秒表一样均匀增加。比如它每秒涨 1，如果乘 2.0，每秒就涨 2。加在参数里面，会让图像整体产生一种源源不断向左滑动的视觉效果。

齿轮 ③ * 0.5 + 0.5 — 范围映射

sin() 的原始输出在 -1 到 1 之间。对于颜色来说，负数没意义（纯黑），所以必须做缩放和偏移，挤进 0 到 1 的区间。

0.50
0.50

背后原理：乘法改变振幅（把波浪拍扁或者拉大），加法改变位移（整体上移下压）。著名的 * 0.5 + 0.5 恰好能把波谷从 -1 抬到 0，波峰压到 1，实现完美的归一化。

齿轮 ④ 组装后的效果

四个齿轮组装后，wave 是一个 0~1 的值，在空间上形成横向条纹（因为用了 uv.x），并且随时间不断向左滚动（因为加了 u_time）。

第三步：用波浪值驱动颜色混合

vec3 color = mix(colorA, colorB, wave);

我们已经知道 mix(A, B, t) 是按比例混合两种颜色。现在 t 不再是固定数字，而是那个在 0~1 之间不停跳动的 wave！

  • wave = 0 的地方 → 显示深蓝色 colorA (0.1, 0.1, 0.3)
  • wave = 1 的地方 → 显示青绿色 colorB (0.0, 0.8, 0.6)
  • wave 在 0~1 之间的地方 → 两种颜色按比例混合！
  • 因为 wave 随时间不断变化，所以颜色也在不断涌动 🌊

第四步：叠加第二层波浪（更丰富）

float wave2 = sin(uv.y * 8.0 - u_time * 1.5) * 0.5 + 0.5;

color = mix(color, colorC, wave2 * 0.4);

💡 关键对比
第一层用 uv.x → 波浪沿水平方向变化 → 画面呈竖条纹（同列同色）
第二层用 uv.y → 波浪沿垂直方向变化 → 画面呈横条纹（同行同色）
两层叠加后，一横一竖交织出复杂的网格涌动效果。wave2 * 0.4 让第二层只有 40% 的影响力，像一层半透明滤镜。

🧩 进阶追问：为什么混合强度 1.0 时粉色带宽度不均匀？

当你把 wave2 * 0.4 改成 wave2 * 1.0（满强度），会发现粉色带在某些列看起来更宽，某些列更窄。但数学上，wave2 只和 uv.y 有关，和 uv.x 完全无关！宽度应该完全一样才对啊？

答案：数学宽度相同，感知宽度不同

这是一个视觉感知错觉（Mach Band 效应的近亲）。让我们看看 wave2 = 0.5（过渡区中间）时发生了什么：

底色列 底色 RGB mix(底色, 粉色, 0.5) 结果 人眼感知
青绿列 (0, 204, 153) (127, 139, 140) 灰雾色，不太像粉色
深蓝列 (25, 25, 76) (140, 52, 89) 暗红/粉色，明显是粉！

本质原因：

  • 深蓝底色亮度很低 → 只需要一点点粉色混入，整体亮度就会被粉色主导 → 粉色"提前显现"
  • 青绿底色亮度很高 → 需要更多粉色才能在视觉上"盖过"底色的光谱 → 粉色"延迟显现"

所以看起来：深蓝区域的粉色带更宽（粉色早出现晚消失），青绿区域更窄（粉色晚出现早消失）。

💡 Shader 实战意义
这就是为什么代码用 wave2 * 0.4 而不是 wave2 * 1.0：
① 低强度混合让两层波浪的"叠加"更均匀，不会出现明显的宽度差异
② 保留底层颜色的辨识度，让画面层次更丰富
③ 避免第二层完全覆盖第一层导致信息丢失

这是 Shader 中alpha blending的核心哲学：不是"覆盖"，而是"叠加"。
