🖱️ NDC — 从屏幕像素到 Shader 的「通用语言」

浏览器告诉你鼠标在 第几个像素，Three.js Raycaster 和全屏后处理却需要 NDC（Normalized Device Coordinates）——范围 [-1, 1]、原点在画面中央的标准坐标。

一、三种坐标系，一张图看懂

坐标系原点Y 轴方向典型范围
屏幕像素 clientX/Y左上角向下 ↑[0, width] × [0, height]
归一化 UV左下角向上 ↑[0, 1] × [0, 1]
NDC画面中心向上 ↑[-1, 1] × [-1, 1]

NDC 与 WebGL 裁剪空间一致：Raycaster 用它发射射线，后处理 Shader 用它定位鼠标轨迹。

二、转换公式：两步走

先归一化到 [0, 1]，再映射到 [-1, 1]，最后翻转 Y 轴

Step 1：像素 → 归一化 [0, 1]

float nx = (clientX - rect.left) / rect.width;
float ny = (clientY - rect.top)  / rect.height;

减去 rect.left/top 是因为 canvas 可能不在窗口左上角。

Step 2：归一化 → NDC [-1, 1]

ndc.x =  nx * 2.0 - 1.0;
ndc.y = -(ny * 2.0 - 1.0);   // Y 取反！

公式与 uv × 2 − 1 同源（见 UV 坐标变换页）。Y 取反是因为屏幕 Y 向下、NDC Y 向上。

四、Dracarys 案例中的用法

mouseNDC 驱动交互

Dracarys 案例（dracarys-02-ndc-mouse）每帧把鼠标位置写入 uniform mouseNDC：

uniform vec2 mouseNDC;  // 范围 [-1, 1]

// 龙身朝向鼠标
vec2 dir = normalize(mouseNDC - dragonPos);
// 后处理：鼠标轨迹光晕
float glow = exp(-length(uv - mouseNDC) * 3.0);

同一套 NDC 坐标同时服务 3D 射线拾取和2D 后处理，是案例交互的核心桥梁。
