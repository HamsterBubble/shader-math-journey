❓ 代码彻底拆解：smoothstep 对比实验

这节课的代码在一个屏幕上同时展示了"硬切 step"和"平滑 smoothstep"的视觉对比，并在中间画了一条金色分割线。我们分四步把每一行都嚼碎。

第一步：给每个像素发身份证 (UV 坐标)

vec2 uv = gl_FragCoord.xy / u_resolution.xy;

屏幕是由无数个小方块（像素）组成的。GPU 调用这段代码时，每个像素会独立运行一次。gl_FragCoord 是当前像素的"门牌号"（比如第 400 列、第 300 行），除以屏幕总尺寸后就变成了 0 到 1 的标准坐标。

  • uv.x = 0 → 像素在屏幕的最左边
  • uv.x = 0.5 → 正中间
  • uv.x = 1 → 最右边
  • uv.y 同理，从底到顶 0→1

第二步：两种截然不同的"横向过渡"

float smooth = smoothstep(0.4, 0.6, uv.x);

float hard = step(0.5, uv.x);

smoothstep：柔化渐变

在 uv.x = 0.4 到 uv.x = 0.6 之间，值从 0 平滑地爬升到 1。就像一个缓缓上升的斜坡。

step：一刀切

在 uv.x = 0.5 这里，瞬间从 0 跳到 1。完全没有过渡，像用裁纸刀切开。

这两个值此刻只是两个"方案备选"——还没有被用上画面。接下来用 mix 决定谁出现在上半屏、谁出现在下半屏。

第三步：万能遥控器 — mix + step

float result = mix(hard, smooth, step(0.5, uv.y));

这是全课最妙的一行！拆成嵌套逻辑：

  • step(0.5, uv.y)：如果像素在屏幕下半部分（uv.y < 0.5）→ 返回 0；上半部分（uv.y ≥ 0.5）→ 返回 1。
  • 这个 0 或 1 被当做 mix 的 t 值传入。
  • t = 0 → mix(hard, smooth, 0) = 100% hard。下半屏显示硬切效果！
  • t = 1 → mix(hard, smooth, 1) = 100% smooth。上半屏显示平滑效果！

第四步：用减法雕刻出分界线

float line = smoothstep(0.498, 0.5, uv.y) - smoothstep(0.5, 0.502, uv.y);

两个 smoothstep 相减 = 一根极细的脉冲线条。只有在 uv.y ≈ 0.5 极小的范围内，两个 smoothstep 的重叠错位区才会产生非零值。其他地方因为两者相减结果为 0 而完全透明。

蓝色是第一个 smoothstep，红色是第二个，绿色是它们的差值（就是那根线！）

第五步：色彩叠加 — 给灰度注入灵魂

vec3 color = vec3(result * 0.3, result * 0.7, result);

color += vec3(1.0, 1.0, 0.0) * line * 0.5;

第一行：把灰度变成冷色调

result 是个 0~1 的灰度值。乘以不同的系数给 RGB 三通道，红光被压到 30%，绿光给了 70%，蓝光保持 100%。于是原本的黑白渐变变成了暗青蓝色的冷色调渐变。

第二行：用 += 叠加金色分割线

这一行拆成四个步骤理解：

  1. vec3(1.0, 1.0, 0.0) → 配色：纯黄色（红+绿满，蓝为零）
  2. * line → 遮罩裁剪：line 只在中间那根细线处有值，其它地方全是 0。乘以 0 = 完全消失。所以黄色只出现在线条上！
  3. * 0.5 → 降低亮度：不想太刺眼，打个五折
  4. color += → 光效叠加：用 += 而不是 =，让黄色像灯光一样叠加在底色之上，交汇处变得更亮更暖！
